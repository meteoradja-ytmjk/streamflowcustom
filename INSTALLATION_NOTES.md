# Catatan Instalasi untuk Windows

## Perubahan yang Dilakukan

### 1. Session Store
Aplikasi ini telah dimodifikasi untuk mengatasi masalah kompatibilitas `connect-sqlite3` dengan `sqlite3` versi terbaru:

**Perubahan:**
- Mengganti `connect-sqlite3` dengan `better-sqlite3-session-store`
- Menggunakan `better-sqlite3` sebagai engine database untuk session

**File yang dimodifikasi:**
- `app.js` - Updated session store configuration

### 2. Dependencies
Dependencies yang telah diperbarui untuk kompatibilitas Windows:

```json
{
  "better-sqlite3": "^11.x.x",
  "better-sqlite3-session-store": "^0.1.x",
  "sqlite3": "^5.1.7"
}
```

### 3. Environment Configuration
File `.env` telah dibuat dengan konfigurasi berikut:

```env
PORT=7575
SESSION_SECRET=<auto-generated-secret>
NODE_ENV=development
```

## Instalasi di Windows

### Prasyarat
- Node.js v18 atau lebih baru
- npm atau yarn
- Visual Studio Build Tools (untuk kompilasi native modules)

### Langkah Instalasi

1. **Clone repository**
   ```cmd
   git clone https://github.com/meteoradja-ytmjk/streamflowcustom.git
   cd streamflowcustom
   ```

2. **Install dependencies**
   ```cmd
   npm install --legacy-peer-deps
   ```

3. **Generate session secret**
   ```cmd
   node generate-secret.js
   ```

4. **Jalankan aplikasi**
   ```cmd
   npm start
   ```

5. **Akses aplikasi**
   Buka browser dan akses: `http://localhost:7575`

## Troubleshooting Windows

### Error: python not found
Install Python dan tambahkan ke PATH, atau install windows-build-tools:
```cmd
npm install --global windows-build-tools
```

### Error: MSBuild not found
Install Visual Studio Build Tools dari Microsoft

### Port sudah digunakan
Ubah PORT di file `.env`:
```env
PORT=8080
```

## Production Deployment

Untuk production di Windows Server, gunakan PM2:

```cmd
npm install -g pm2
pm2 start app.js --name streamflow
pm2 save
pm2 startup
```

## Catatan Penting

1. **Session Database**: Session disimpan di `db/sessions.db`
2. **Main Database**: Database utama di `db/streamflow.db`
3. **Uploads**: File upload tersimpan di `public/uploads/`
4. **Logs**: Application logs di folder `logs/`

Semua folder ini sudah ditambahkan ke `.gitignore` untuk keamanan.
